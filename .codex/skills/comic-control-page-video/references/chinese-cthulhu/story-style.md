# 中式克苏鲁原创怪谈规范

用于 `comic-control-page-video` 的中式克苏鲁怪谈模式。目标是生成原创小说或可转口播的怪谈稿，不写历史讲解，不依赖真实历史人物、真实朝代事件或考据底座。

## 核心定位

- 类型：中式民俗恐怖 + 不可名状恐怖 + 认知污染。
- 场景：虚构村镇、河湾、祠堂、古井、祖屋、庙、义庄、废弃厂房、旧县志库房、家族坟山、盐碱地、潮湿地下室。
- 素材：族谱、牌位、红线、朱砂、香灰、井水、黑潮、盐霜、旧历、禁忌日、无名碑、残页、梦游、祖训、地方传说。
- 怪异来源：地底、井下、河眼、祖坟裂隙、旧庙壁画、族谱空白页、地下潮声。可以出现虚构古神、旧神、邪神、触须和巨型怪物，但不要解释成完整神话体系或怪物图鉴。

## 禁区

- 不写真实历史人物、真实朝代斗争、真实战役、真实宫廷事件。
- 不写成历史科普、地方志考据、朝代复原或“古代某身份到底多爽 / 多惨”。
- 不写成游戏机制说明、战力等级、技能表或怪物克制表。
- 可以写古神、触须、巨眼、黑水和怪物实体，尤其当用户明确要求时；但恐怖不能只靠“触手很多”，必须和中式民俗空间、仪式、名字、族谱、香火、井水或认知污染绑定。
- 不直接模仿具体在世作者的个人文风；只使用类型特征。

## 叙事结构

1. 日常入口：主人公因搬家、回乡、采访、整理遗物、修井、参加丧事、查账、修祠堂等进入异常现场。
2. 第一处错位：声音、气味、水质、文字、影子、梦或亲人记忆出现轻微错误。
3. 民俗规则：老人、族长、家规、祖训或旧纸条给出一句禁忌；规则要短，不要解释完整。
4. 违规或被迫接近：主人公为现实利益、亲情、工作或求真继续靠近异常。
5. 认知污染：名字、族谱、照片、影子、时间、空间方向、人际记忆开始错位。
6. 不可名状露面：只露出局部和关系，不完整描写本体；重点写“人理解不了”带来的崩塌。
7. 代价结尾：主人公失去名字、记忆、位置、亲属关系或现实身份；结尾留下扩散迹象。

## 语言和节奏

- 句子要清楚、冷、具体。用动作和物件推进，不连续堆形容词。
- 每 150 到 250 字落一次可见细节：潮味、铁锈、香灰、纸背霉斑、井沿盐霜、灯油、皮肤湿冷、门缝黑水。
- 恐怖升级要慢：先不对劲，再不合理，再无法解释，最后不可承受。
- 不要让人物频繁大喊或解释世界观。人越慌，文字越克制。
- 结尾不要全解释。保留一个能让图片继续扩展的视觉钩子。

## 必须产出的审核信息

`story_package.json` 建议包含：

```json
{
  "version": 1,
  "mode": "chinese_cthulhu_weird_tale",
  "title": "井底天",
  "first_sentence": "我第一次听说柳湾村，是在一本残缺的县志里。",
  "target_word_count": "2200-3200",
  "narration_path": "01_script/narration.md",
  "review_packet_path": "01_script/human_review_packet.md",
  "fiction_boundary_path": "01_script/fiction_boundary.md",
  "comment_question": "如果你在井里听见有人叫出你的本名，你会回应吗？",
  "image_keywords": ["古井", "祠堂", "族谱", "黑潮", "盐霜", "无眼神像"]
}
```

`fiction_boundary.md` 必须说明：

- 本稿为原创虚构怪谈。
- 不使用真实历史人物和真实历史事件作为事实底座。
- 民俗、村名、族谱、禁忌和仪式均为虚构或混合化设定。
- 若使用参考图或文字 prompt，只作为气氛和视觉方向，不复刻具体作品。

## 自检

- 是否误写成历史讲解或朝代考据。
- 是否过早解释怪物来历；古神或怪物可以露面，但不要把它写成设定百科。
- 是否每个恐怖升级都有具体物件或动作承接。
- 是否让不可名状存在保持“局部可见、整体不可懂”。
- 是否留下适合 9:16 单页剧情图生成的场景节点、人物动线和视觉关键词。
